# Penetration Testing Research Papers — Comparative Analysis

| Dimension | Paper 1 — LSTM-ESNet VAPT (Shaik & Jabez · IEEE ICUIS 2025) | Paper 2 — Kali Linux Pen-Testing (Kumar et al. · IEEE WCONF 2023) | Paper 3 — LLMs for PenTest Education (Nizon-Deladoeuille et al. · IEEE SNAMS 2024) |
|-----------|--------------------------------------------------------------|-------------------------------------------------------------------|--------------------------------------------------------------------------------------|
| **Focus Area** | Deep learning-based vulnerability detection in hybrid networks via VAPT | Empirical pen-testing using Kali Linux tools on real-world targets | Evaluating LLMs as educational aids for penetration testing students |
| **Core Methodology** | Hybrid LSTM-ESNet model; LSTM captures temporal patterns, ESNet handles non-linear high-dimensional data. Trained on MegaVul dataset (320,000+ functions) | Experimental approach using ZAP Proxy, Cookie Editor, Kali Linux inside VMware. Demonstrated broken authentication exploit on lucidmotors.com (bug bounty target) | 15 pen-testing tasks across 6 LLMs (GPT-4o, GPT-4o mini, Gemini, Llama, Mixtral, WhiteRabbitNeo) against Metasploitable v3 and OWASP WebGoat |
| **Tools & Technologies** | Python, LSTM, Echo State Network (ESNet), Kaggle/MegaVul Dataset | Kali Linux, ZAP Proxy, CookieEditor, VMware, Firefox, Chrome | GPT-4o mini, GPT-4o, WhiteRabbitNeo, Gemini 1.5 Flash, Llama 3.1 405B, Mixtral 8x7B |
| **Key Results** | 95.64% accuracy at epoch 5; 97.41% at epoch 25. Outperformed RNN, Bi-GRU, LSTM, ESNet baselines. Precision 13.26% higher than RNN. Lowest compute time (46.7 min) and memory (201 kb) | Successfully exploited broken authentication & session management on a live site. ZAP Proxy produced many false positives before finding valid exploit. Demonstrated cookie-based session hijacking | GPT-4o mini & GPT-4o: best success rate (13/15 tasks). WhiteRabbitNeo: 11/15, highest creativity. Gemini: lowest (5/15) due to stringent restrictions. GPT-4o mini best for education |
| **Evaluation Metrics** | Accuracy, F1-Score, FDR, FNR, FOR, MCC, NPV, Precision, PT, Sensitivity | Qualitative exploit success; false positive rate from ZAP Proxy scan | Length, Relevance, Usability, Explanation, Restrictions, Variety, Creativity (7-criteria rubric) |
| **Limitations** | May fail on unseen/adversarial attacks not in training data. Dataset limited to C/C++/Java. No explainability in decision-making | High false positive rate from ZAP Proxy. Cannot detect zero-day vulnerabilities. Limited scope — single exploit type. Duplicate reporting issue encountered | Small LLM sample. Single-response evaluation limits robustness. Performance varies by prompt structure. Results may not generalise across all pen-testing contexts |
| **Research Gap Addressed** | Conventional ML struggles with sequential, non-linear cybersecurity data in hybrid networks; high false positives and slow response times | Lack of practical, hands-on demonstrations of pen-testing workflows using freely available OS-level tools | No prior study comparing multiple LLMs specifically for penetration testing education effectiveness |
| **Future Scope** | Integrate Explainable AI (XAI) for transparent decision-making. Extend to more network types. Handle adversarial attacks not in training data | Develop tools to reduce false positives. Expand to zero-day and emerging threat detection. Create new methodologies for novel attack vectors | Test on larger student cohorts. Evaluate newer/domain-specific LLMs. Multi-response evaluation per prompt. Investigate prompt engineering strategies |
